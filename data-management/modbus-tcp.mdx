## What is Modbus TCP?

Modbus TCP is an industrial communication protocol that enables data acquisition from PLCs, sensors, and other field devices over Ethernet networks using TCP/IP. It standardizes how discrete and register data are read and written, facilitating interoperability between diverse hardware and software systems.

## Device Profile Configuration

### Connection Parameters

Configure the Modbus TCP connection settings in your Proxus Device Profile:

| Parameter            | Description                                 | Default | Format                 | Required |
| -------------------- | ------------------------------------------- | ------- | ---------------------- | -------- |
| **Host**             | IP address or hostname of the Modbus device | -       | IPv4 or DNS name       | Yes      |
| **Port**             | TCP port for Modbus (usually 502)           | 502     | 1–65535                | No       |
| **ReadingFrequency** | Polling interval in milliseconds            | 1000    | 100–60000              | No       |
| **ConnectTimeOut**   | Connection timeout in milliseconds          | 1000    | 500–30000              | No       |
| **ReceiveTimeOut**   | Read response timeout in milliseconds       | 1000    | 500–30000              | No       |
| **DataFormat**       | Byte order for multi-register values        | ABCD    | ABCD, BADC, CDAB, DCBA | No       |

#### DataFormat Options

* **ABCD**: Word1 high → Word2 low (default)
* **BADC**: Word1 low → Word2 high
* **CDAB**: Byte-swap each word high → low
* **DCBA**: Byte-swap each word low → high

## Tag Configuration

Each tag in Proxus represents a Modbus data point. Below are the key settings:

### Basic Tag Fields

1. **Name**: Friendly tag identifier
2. **Address**: Modbus address string
3. **Data Type**: Type of the data (Bool, Int16, Float, etc.)
4. **Length**: Number of registers (only for String/Bytes)
5. **Polling Interval**: Overrides device default if >0

### Address Format

```
[s=station];[x=function];address[.bit]
```

| Component   | Description                      | Example |
| ----------- | -------------------------------- | ------- |
| **s=<ID>**  | Slave ID (1–247)                 | s=2     |
| **x=<FC>**  | Modbus function code             | x=4     |
| **address** | Register or coil number          | 100     |
| **.bit**    | Bit index within register (0–15) | .5      |

### Function Codes and Prefixes

| FC | Operation                | Address Prefix       | Description                   |
| -- | ------------------------ | -------------------- | ----------------------------- |
| 01 | Read Coils               | `x=1;`               | Read discrete output coils    |
| 02 | Read Discrete Inputs     | `x=2;`               | Read discrete input contacts  |
| 03 | Read Holding Registers   | (default)            | Read 16-bit holding registers |
| 04 | Read Input Registers     | `x=4;`               | Read 16-bit input registers   |
| 05 | Write Single Coil        | (via Write Bool)     | Force single coil             |
| 06 | Write Single Register    | (via Write Int16)    | Write single 16-bit register  |
| 15 | Write Multiple Coils     | (via Write Bool\[])  | Force multiple coils          |
| 16 | Write Multiple Registers | (via Write Int16\[]) | Write multiple registers      |

### Address Examples

| Address String | Operation                       | FC |
| -------------- | ------------------------------- | -- |
| `100`          | Holding register at 100         | 03 |
| `s=2;100`      | Holding register 100 on slave 2 | 03 |
| `x=4;100`      | Input register at 100           | 04 |
| `s=2;x=4;100`  | Input register 100 on slave 2   | 04 |
| `100.0`        | Bit 0 of holding register 100   | 03 |
| `x=2;100`      | Discrete input 100              | 02 |
| `x=1;100`      | Coil 100                        | 01 |

## Supported Data Types

### Fixed-Size Types

| Type       | Registers | Bytes | Description                   |
| ---------- | --------- | ----- | ----------------------------- |
| **Bool**   | -         | 1 bit | Single coil or discrete input |
| **Short**  | 1         | 2     | Signed 16-bit integer         |
| **UShort** | 1         | 2     | Unsigned 16-bit integer       |
| **Int**    | 2         | 4     | Signed 32-bit integer         |
| **UInt**   | 2         | 4     | Unsigned 32-bit integer       |
| **Float**  | 2         | 4     | IEEE 754 single-precision     |
| **Long**   | 4         | 8     | Signed 64-bit integer         |
| **ULong**  | 4         | 8     | Unsigned 64-bit integer       |
| **Double** | 4         | 8     | IEEE 754 double-precision     |

### Variable-Length Types

| Type       | Description        | Length Field Use    |
| ---------- | ------------------ | ------------------- |
| **String** | ASCII/Unicode text | Number of registers |
| **Bytes**  | Raw binary data    | Number of registers |

## Advanced Features

### Bit-Level Access

Add `.bit` to read a specific bit:

```
Address: 100.5
Data Type: Bool
```

Reads bit index 5 of holding register 100.

### Overriding Data Format

Use `format=` prefix to change byte order:

```
Address: format=BADC;s=1;100
Data Type: Float
```

Reads float with BADC ordering.

## Best Practices

* **Group sequential addresses** for efficient batch reads.
* **Adjust polling** based on data volatility.
* **Use correct data types** and **exact lengths** for String/Bytes.
* **Prefix station** (`s=`) only when multiple slaves share a gateway.

## Troubleshooting

| Issue                     | Cause                                 | Solution                                 |
| ------------------------- | ------------------------------------- | ---------------------------------------- |
| Data read out of bounds   | Requested registers exceed device map | Verify Length and available registers    |
| Invalid address           | Address not supported by device       | Check device memory map                  |
| Unsupported function code | Device firmware lacks FC support      | Try alternate FC or update firmware      |
| Byte order mismatch       | Wrong DataFormat setting              | Test with ABCD, BADC, CDAB, DCBA options |
