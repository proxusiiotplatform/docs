# Quick Start Guide

This guide will help you set up Proxus and configure your first data collection pipeline. Follow these steps to quickly experience the power of Proxus.

## Prerequisites

Before you begin, ensure you have:

<Check>
  - Docker installed on your system
  - Access to an industrial device or simulator
  - Network connectivity to your data source
  - Minimum 4GB RAM and 2 CPU cores available
</Check>

## Step 1: Deploy Proxus

```bash
# Pull the Proxus Docker image
docker pull proxus/platform:latest

# Create a persistent volume for TimescaleDB
docker volume create proxus-timescaledb-data

# Start Proxus using Docker Compose
docker-compose up -d
```

<Note>
Default credentials for first-time login:
- Username: `admin`
- Password: `proxus123`

**Important:** Change these credentials immediately after your first login.
</Note>

## Step 2: Access the Platform

1. Open your web browser
2. Navigate to `http://localhost:8080`
3. Log in using the default credentials
4. Complete the initial setup wizard

## Step 3: Configure Your First Device

### Create a Device Profile

<Steps>
  <Step title="Navigate to Device Profiles">
    Click on "Configuration" → "Device Profiles" in the main menu
  </Step>
  
  <Step title="Add New Profile">
    Click the "+" button to create a new device profile
  </Step>
  
  <Step title="Configure Profile">
    ```json
    {
      "name": "ModbusDevice",
      "protocol": "ModbusTCP",
      "settings": {
        "ipAddress": "192.168.1.100",
        "port": 502,
        "unitId": 1,
        "scanRate": 1000
      }
    }
    ```
  </Step>
</Steps>

### Configure Tags

<Frame>
  ```mermaid
  graph LR
    A[Create Tag] --> B[Set Address]
    B --> C[Define Data Type]
    C --> D[Set Scan Rate]
    D --> E[Enable Tag]
    style A fill:#d4e6ff,stroke:#333,stroke-width:2px
    style B fill:#d4e6ff,stroke:#333,stroke-width:2px
    style C fill:#d4e6ff,stroke:#333,stroke-width:2px
    style D fill:#d4e6ff,stroke:#333,stroke-width:2px
    style E fill:#d4e6ff,stroke:#333,stroke-width:2px
  ```
</Frame>

1. Navigate to "Tags" section
2. Click "Add New Tag"
3. Configure basic tag settings:
```json
{
  "name": "Temperature",
  "address": "40001",
  "dataType": "Float",
  "scanRate": 1000,
  "enabled": true
}
```

## Step 4: Create Your First Rule

### Set Up Real-time Analysis

<Steps>
  <Step title="Navigate to Rules">
    Open "Rules Engine" from the main menu
  </Step>
  
  <Step title="Create Rule">
    Create a simple temperature threshold rule:
    ```csharp
    if (Tag["Temperature"].Value > 30) {
        Alert("High Temperature", $"Temperature exceeded: {Tag["Temperature"].Value}°C");
    }
    ```
  </Step>
  
  <Step title="Activate Rule">
    Enable the rule and set its execution priority
  </Step>
</Steps>

## Step 5: Build a Dashboard

### Create Visualization

<Steps>
  <Step title="Open Dashboard Designer">
    Navigate to "Dashboards" → "New Dashboard"
  </Step>
  
  <Step title="Add Components">
    Drag and drop these components:
    - Line Chart (for temperature trend)
    - Current Value Display
    - Alert History
  </Step>
  
  <Step title="Configure Data Source">
    ```sql
    SELECT time, value 
    FROM tags.temperature 
    WHERE time > now() - interval '1 hour'
    ```
  </Step>
</Steps>

## Step 6: Verify Data Flow

1. Monitor the "Live Data" view
2. Check TimescaleDB storage:
```sql
SELECT * FROM tags.temperature 
ORDER BY time DESC 
LIMIT 10;
```
3. Verify rule triggers in the alert history

## Troubleshooting Common Issues

<Accordion title="Can't connect to the device?">
  1. Verify network connectivity (ping device IP)
  2. Check firewall settings
  3. Confirm device protocol settings
  4. Review adapter configuration
</Accordion>

<Accordion title="No data in dashboard?">
  1. Verify tag configuration
  2. Check device connection status
  3. Confirm SQL query syntax
  4. Review time range settings
</Accordion>

<Accordion title="Rules not triggering?">
  1. Check rule syntax
  2. Verify rule is enabled
  3. Review trigger conditions
  4. Check tag data quality
</Accordion>
